## 二、项目目标（客户可感知价值）

1. **主动守护**：用户手动登记药箱；FDA 召回匹配后通过**邮件**、**站内通知**（阶段一，见 M8）、**短信**（完整版）提醒。
2. **随时核查**：手动输入药品信息查询召回；未登录用户 **限 2 次**查询，之后须注册。
3. **商业化**：免费监控 **2 个药品**；更多药品或家庭场景通过 **订阅支付**（服务商待确认，定价见第五节）。
4. **风险分级**：Class I / II / III **不同通知模板与界面样式**（参照 FDA 官方呈现）。
5. **合规透明**：仅信息聚合与提醒，不提供用药或停药建议。
6. **数据时效**：FDA 数据约 **每周**更新；匹配召回后，用户在 **24 小时内**收到已启用渠道的通知（服务等级见下表）。

**服务等级（SLA）**：


| 项目         | 约定                                            |
| ---------- | --------------------------------------------- |
| FDA 召回数据同步 | 约 **每周** 增量/全量同步                              |
| 用户通知触达     | 匹配成功后 **24 小时内** 通过邮件/站内/短信（以用户偏好与 opt-in 为准） |

**本阶段不包含**：短信、浏览器/Web Push、家庭药箱、订阅支付闭环（可先做第 3 个药拦截提示）、管理后台。

### 3.2 阶段一 — MVP 验收清单

- 召回数据可同步并显示更新时间  
- 注册含必填资料项；Google 登录可用  
- 未登录用户仅可查询 **2 次**，之后须注册  
- 药箱手动维护；最多 **2 个**免费监控药品；删除后不再追踪  
- 未知厂商提示「无法追踪」 
- 监控期内匹配召回可收邮件与站内通知  
- 手动即时查询可用（无扫码、拍照、短信、浏览器/Web Push）  
- 法律静态页与 Cookie 说明已上线  
- MVP 环境稳定、同步按日执行
